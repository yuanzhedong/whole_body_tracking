BFM-Zero → Control-VAE distillation

Reference motion → humanoid action. Design + live experiment status. Offline mirror of the W&B report.

What this is. Building the faithful BeyondMimic Fig-7 control VAE, but distilling BFM-Zero (the Forward-Backward foundation-model tracker) as the single generalist teacher. The VAE maps reference → action, with an N(0,I) latent, trained on (state, reference, action) triples under domain randomization, validated sim2sim in MuJoCo. Goal: fuse the two-stage generate-then-track pipeline into one latent→action controller with a diffusion-ready latent.

Why distill BFM-Zero (vs the validated generate-then-track pipeline)

  1. One deployable controller — a = D(z, proprio) replaces decode-then-track.
  2. A diffusion-ready latent — BFM-Zero's z lives on a √d-sphere (project_z), not a Gaussian; the KL term gives an N(0,I) action-latent, which downstream latent-diffusion needs (raw actions have torque spikes that break diffusion — the VAE smooths them).
  3. Inherit BFM-Zero's coverage — including near-ground crouch/sit/squat that HoloMotion collapses on — in a compact student.

Key insight — BFM-Zero already factorizes as a control VAE

Verified in humanoidverse/agents/fb/model.py:

z = model.backward_map(ref_obs)     # reference/goal motion -> latent z   (E)
a = model.act(proprio_obs, z)       # z + current state     -> action     (D)

So we distill the pair (backward_map, act) into a VAE with a Gaussian bottleneck. Interface (dims verified from a live rollout of the pretrained model):

Action = position targets (not torque) ⇒ behaviour-cloning is well-behaved (no spikes).

Model, loss, and the encoder horizon H

MotionVAE (stage2/vae_model.py): E(ref_window[H]) → (μ,logσ), z~N(μ,σ); D(z, proprio) → â. State-conditioning is non-negotiable (under DR/noise the same reference needs different corrections in different states).

Hregimenote
1per-step causal (Fig-7 exact)most reactive
~8–16short horizon (recommended)16 aligns with the Fig-7 diffusion horizon
≥64long / whole-clipgood for generation, too long for control

H is a training-time knob: the collector logs the per-step reference, so any H window is a cheap train-time slice (sweep without re-collecting). Distinct from the motion-reconstruction VAE's 128-frame window (that length is right for generation).

The hard problem — covariate shift & the two roles of domain randomization

The student is the tracker now, so its own errors feed back; naive offline BC drifts to states BFM-Zero never visited → falls. Escalating fixes: (1) DART / noise-injection BC — step the env with a noised action to fatten the visited-state tube, but label each state with the clean teacher action a*; (2) DAgger — roll out the student, relabel with BFM-Zero; (3) on-policy + OU noise + DR.

DR has two roles: (a) robustness for sim2sim/sim2real (mass/friction/motor/latency + obs noise); (b) covariate coverage — DR widens the collected-state distribution. Subtlety: BFM-Zero was trained with DR, so match its ranges first, then widen.

⚠ Key finding (decision-guiding): the latent collapses. Across all 72 configs, the max z-ablation is 0.016 — far below the G3 gate of 0.1 — while reconstruction is uniformly good (val RMSE ≈ 0.10–0.11 in action units). No hyperparameter rescues it. Interpretation: the 929-D proprio observation already determines BFM-Zero's action, so the decoder reconstructs from proprio alone and ignores the latent (posterior collapse from over-informative conditioning). The current formulation degenerates to a proprio→action clone (a conditional policy), not a useful generative latent.

Weak trends: bigger latent + align=0.5 + low β give marginally higher z-ablation (0.016 vs 0.011) — but all negligible. The signal is unambiguous: proprio dominates.

➡ Recommended next step (phase 2b). Force the reference to flow only through the encoder→z: strip the goal/reference-tracking terms from the decoder's proprio input so the decoder sees true proprioception (joint pos/vel, base, gravity) but not the reference. If z-ablation then rises above 0.1, the latent is carrying motion intent and the VAE is generative-ready. If reconstruction collapses instead, it confirms the action is fundamentally state-reactive and we accept a conditional-policy latent (fine for control, weak for diffusion) — either way it decides the Phase-3 direction. Cheap to test: it's an input-mask ablation on the existing pairs, no re-collection.
